# Coreless 새로운 40차 제작 — 7차

## 선행 확인: 6차 실제 화면과 전투 스트레스 검사

6차 경기장을 Chromium에서 다시 실행하고, 세 번째 파동에서 적 두 기를 같은 위치에 놓는 결정적 충돌 검사를 실시했다.

- 6차에서는 같은 위치에 놓인 두 적이 다음 물리 프레임에도 38px만큼 겹친 상태를 유지했다.
- 플레이어를 적 사이에 360프레임 동안 세워 둔 접촉 스트레스 검사에서는 누적 피해로 1회 사망했다.
- 여러 적이 한 지점에 모이면 서로 밀어내지 않고, 플레이어가 무적 시간이 끝날 때마다 연속 피해를 받을 수 있었다.

## 7차 선행 수정

- 적 중심 간 최소 분리 기준을 58px로 설정했다.
- 이동 방향 보정과 최종 충돌 분리의 두 단계로 적 겹침을 제거했다.
- 적 접촉 피해 뒤 전체 적에게 공통으로 적용되는 132프레임의 접촉 유예 시간을 추가했다.
- 같은 조건의 재검사에서 강제로 겹친 적이 한 프레임 안에 분리되었다.
- 360프레임 접촉 스트레스 검사에서 피해는 발생했지만 사망하지 않았다.

## 승강 전투 통로 구현

- 공간 범위: x=11700~14700, 폭 3000px
- 2160px 낭떠러지 위로 이어지는 하나의 공중 경로
- 수평 이동 발판 3개와 수직 이동 발판 2개
- 이동 발판 폭 190~220px
- 발판 위에 서 있으면 발판의 x·y 이동량이 플레이어에게 그대로 전달되는 탑승 보정
- 고정 전투 발판 2개와 마지막 착지 발판
- 낭떠러지에 떨어지면 입구 체크포인트로 즉시 복귀

### 구간별 적 배치

- 첫 번째 고정 발판: 굴착충 1기
- 중앙 공중 구간: 부유 감시체 1기
- 마지막 착지 발판: 기계 사냥개 1기
- 적은 플레이어가 520px 안으로 접근했을 때만 적극적으로 추적한다.
- 세 적의 크기는 38~48px로 플레이어와 비슷하거나 약간 큰 범위를 유지한다.

## 진행 구조

- 입구를 지나면 입구문과 출구문이 잠긴다.
- 세 적을 모두 처치하면 양쪽 잠금이 해제되고 오른쪽 출구가 열린다.
- 이동 발판 이외의 우회용 바닥은 존재하지 않는다.
- 입구 체크포인트와 종료 체크포인트를 분리해 배치했다.

## 실제 Chromium 검사

### 실제 키보드 입력

- 실제 D 입력으로 x=11825에서 약 12022.91까지 이동하고 혼합 구역 시작·문 잠금을 확인했다.
- 실제 J 입력 3회로 첫 번째 굴착충을 처치했다.
- 나머지 두 적은 동일한 피해·사망 함수를 호출하는 테스트 기능으로 처리하여 출구 상태를 결정적으로 확인했다.
- 클리어 후 실제 D 입력으로 `cp_mixed_end`에 도달했다.
- 최종 x는 약 14523.21px였다.

### 이동 발판 탑승 검사

- 첫 번째 수평 이동 발판 위에 플레이어를 배치한 뒤 120프레임을 진행했다.
- 발판과 플레이어 사이의 상대 x좌표 오차는 0px였다.
- 검사 동안 플레이어는 발판과 함께 약 173.79px 이동했다.

### 전체 상태 검사

- 생성된 적: 3기
- 처리된 적: 3기
- 입구문 잠금 해제: 성공
- 오른쪽 출구 개방: 성공
- 브라우저 콘솔 오류: 0건
- 자동 감사: 14/14 통과
