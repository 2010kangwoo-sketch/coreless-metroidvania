# Coreless 새로운 40차 제작 — 9차

## 8차 연결 상태

9차는 8차 종료 체크포인트 `cp_long_chase_entry`, 이전 실행 상태의 월드 x좌표 18000px 이상 또는 8차 완료 상태를 확인하면 자동으로 시작한다. 수동 검사에서는 F9를 한 번 눌러 바로 진입할 수 있다.

9차가 활성화되면 기존 캔버스를 새 캔버스로 교체한다. 1~8차의 실행 상태는 메모리에 남지만 화면과 입력은 9차 전용 장면으로 전환되므로 두 게임 화면이 같은 캔버스에 동시에 그려지지 않는다.

## 장거리 붕괴석 추격전

- 9차 내부 월드 길이: 6420px
- 실제 추격 진행 거리: 5950px
- 경로를 구성하는 굴곡 기준점: 11개
- 갈림길이나 우회 바닥이 없는 하나의 경사 경로
- 완만한 하강과 상승이 번갈아 이어지는 구조
- 추격 종료 체크포인트: `cp_chase_end`

추격 시작 전 84프레임의 후방 천장 붕괴 경고를 제공한다. 경고가 끝나면 반지름 76px의 붕괴석이 플레이어 뒤에서 낙하해 추격을 시작한다. 플레이어 크기는 34×48px이므로 붕괴석은 플레이어보다 확실히 크지만 길 전체를 막을 정도로 과도하게 크지는 않다.

## 잔해 장애물

- 잔해 장애물: 13개
- 장애물 높이: 38~66px
- 플레이어: 점프로 통과
- 붕괴석: 실제 충돌 후 파괴
- 장애물 충돌 감속: 1.05px/frame
- 감속 유지 시간: 30프레임

잔해는 삼각형에 가까운 실제 외형에 맞춰 충돌 높이가 달라진다. 직사각형 전체를 벽처럼 판정하지 않기 때문에 캐릭터가 정상적인 점프 궤적으로 꼭대기를 넘을 수 있다. 경사면에서 접지 판정이 순간적으로 풀리는 문제를 줄이기 위해 지면 12px 안에서는 점프 입력을 허용하는 짧은 코요테 판정을 추가했다.

## 붕괴석 속도와 긴박감

- 기본 속도: 4.60px/frame
- 최대 속도: 7.65px/frame
- 기준 간격: 520px
- 플레이어가 멀어지면 속도가 증가하고 가까워지면 감소하는 제한된 적응 속도

붕괴석은 장애물을 파괴할 때마다 감속하지만 이후 다시 속도를 회복한다. 플레이어가 대시를 반복해도 돌과의 거리가 계속 무한히 벌어지지 않도록 최대 속도를 높였으며, 반대로 플레이어 바로 뒤에 붙었을 때는 적응 속도가 낮아져 회복 기회를 제공한다.

## 구간 표시와 카메라

- 붕괴문 표시: 3개
- 플레이어가 각 붕괴문을 통과한 뒤 돌이 구조물을 부수며 추가 감속
- 추격 전용 카메라는 플레이어와 붕괴석을 함께 보여 주는 위치를 우선
- 돌과의 거리가 크게 벌어졌을 때는 돌을 화면 왼쪽에 유지
- HUD에 진행도, 돌과의 간격, 남은 잔해, 대시 재사용 시간을 표시

## 압사와 복귀

붕괴석과 충돌하면 일반 피격이 아니라 압사로 처리한다. 낭떠러지 아래로 떨어진 경우에는 추락으로 처리한다. 두 경우 모두 48프레임 뒤 `cp_long_chase_entry`부터 추격이 다시 시작된다. R을 누르면 같은 체크포인트에서 즉시 재시작할 수 있다.

## 실제 Chromium 키보드 완주 검사

Chromium에서 9차 장면을 활성화한 뒤 애니메이션을 고정 프레임으로 진행했다. 이동과 점프, 대시는 Playwright의 실제 키보드 이벤트로 전달했고, 각 입력이 유지되는 프레임을 실제 게임 물리 코드에 적용했다.

- D 입력: 완주할 때까지 유지
- Space 입력: 13회
- Shift 입력: 8회
- 완주 시간: 약 15.12초
- 사망: 0회
- 압사: 0회
- 추락: 0회
- 플레이어가 통과한 잔해: 13개
- 붕괴석이 파괴한 잔해: 11개
- 붕괴석 감속: 11회
- 통과한 붕괴문: 3개
- 최종 체크포인트: `cp_chase_end`
- 브라우저 콘솔 오류: 0건
- 자동 감사: 16/16 통과

완주 순간에는 출구 뒤 차단 구조가 닫히고 붕괴석이 차단 구조에 충돌한 위치로 정리된다. 플레이어가 출구에 도달하자마자 돌이 먼 거리에서 이유 없이 멈추는 화면이 나오지 않도록 했다.

## 실행 구조 검증

`src/pass09.js`는 한 개의 파일이다. 내부 게임 코드를 gzip으로 압축해 포함하고 페이지 시작 시 한 번만 해제한다.

- 외부 데이터 조각 없음
- 재귀 호출 없음
- 자동 재시도 없음
- 실패 시 파일을 다시 생성하는 검증 루프 없음
- 애니메이션 루프 1개
- 8차 완료 확인용 250ms 폴링은 9차 활성화 즉시 해제

압축 실행 파일 자체도 별도의 Chromium 검사에서 `rebuild-pass09`, pass 9, 감사 16/16, 콘솔 오류 0건을 확인했다.
