# Coreless 새로운 40차 제작 — 8차 브라우저 재검증본

## 검증 방식

조직 정책 때문에 Chromium에서 `localhost`, `file:` 및 `data:` 주소가 차단되어 있었다. 따라서 Chromium 자체를 실행한 뒤 DevTools Protocol로 `about:blank` 문서에 동일한 HTML과 JavaScript를 주입했다. 이후 화면 렌더링, 실제 키 입력, 캔버스 스크린샷과 게임 상태를 검사했다. 이는 JavaScript 문법 검사만 한 것이 아니라 실제 Chromium의 렌더링·키보드·애니메이션 환경에서 진행한 검사이다.

## 실제 화면에서 확인한 문제와 수정

### 1. 붕괴석 경고가 너무 일찍 시작됨

기존 재제작본은 구역에 들어가자마자 타이머가 줄어들어 벽점프 구조를 확인하기 전에 압사할 수 있었다. 플레이어 진행도가 900px 이상일 때만 120프레임 경고가 시작되도록 수정했다.

### 2. 대시가 가시밭 끝까지 도달하지 못함

기존 대시는 13프레임으로 첫 가시밭조차 안정적으로 통과하기 어려웠고, 하단 가시밭은 이론상 거리가 부족했다. 대시를 27프레임, 10.4px/frame으로 수정했다. 대시 없이 진입하면 즉시 가시 사망이 발생하고, 대시 한 번으로 각 가시밭을 끝까지 통과할 수 있다.

### 3. 정밀 점프 판정이 실제 조작보다 빡빡함

기존 판정은 파쿠르 구간 안에 있는 모든 프레임에서 높이를 제한해 정상적인 포물선 점프도 중간에 실패했다. 각 구간 진입 순간의 점프 높이로 강도를 판정하고, 두 긴 점프 사이에 다시 착지하고 뛰어오를 수 있는 간격을 확보했다.

### 4. 미로 구조물이 모두 같은 돌무더기로 보임

파괴 구조물을 9개 그룹으로 늘리고 형태를 분리했다.

- 계단식 석벽: 높은 점프로 통과
- 매달린 톱니벽: 낮은 점프로 통과
- 교차 리브: 중간 높이 점프로 통과
- 격자 보강벽: 벽점프 구역과 결합
- 중단 미로벽
- 곡선 보강대
- 하단 계단벽
- 하단 교차벽
- 최종 보강벽

붕괴석은 각 구조물을 실제 진행 경로에서 파괴하며, 그룹 하나를 파괴할 때마다 속도가 1.12px/frame 감소하고 38프레임 동안 감속한다.

## 위에서 아래로 내려가는 스네이크 경로

- 전체 경로 길이: 약 8832.49px
- 시작 위치: 약 (160, 250)
- 첫 번째 완곡 전환: 약 (2888, 425)에서 아래로 떨어져 (2777, 732) 방향으로 좌회전
- 두 번째 완곡 전환: 약 (257, 943)에서 아래로 떨어져 (503, 1267) 방향으로 우회전
- 종료 위치: 약 (2929, 1479)
- 수평 진행 방향: 오른쪽 → 왼쪽 → 오른쪽

바닥 기준점의 높이를 반복적으로 바꿔 각 레인에 오르막, 내리막, 골과 짧은 언덕이 섞이도록 했다. 방향 전환부는 직각 낙하가 아니라 곡선 보간 경로이며, 붕괴석도 플레이어와 같은 경로 진행값을 사용해 곡선을 따라 내려간다.

## 실제 Chromium 키보드 검사

각 기능은 이전 실행의 이벤트가 남지 않도록 매번 새 `about:blank` 문서에서 독립적으로 검사했다.

### 미로벽 점프

- 실제 입력: D 유지 + Space 길게 누르기
- 도달 진행도: 약 968.50px
- 사망: 0회
- 계단식 석벽 통과 확인

### 연속 벽점프

- 실제 입력: A·D를 번갈아 누르며 Space
- 벽점프: 5회
- 종료 진행도: 약 2801.50px
- 사망: 0회
- 벽타기 통과 상태 확인

### 첫 번째 대시 가시밭

- 대시 없이 D만 입력: 가시 사망 1회
- D + Shift 입력: 진행도 약 3954.55px까지 통과
- 대시 사용: 1회
- 사망: 0회

### 점프 강도 파쿠르

- 실제 Space 입력: 2회
- 짧은 점프와 긴 점프를 각각 다른 누름 시간으로 입력
- 종료 진행도: 약 4605.15px
- 사망: 0회

### 낮은 천장 점프

- 실제 입력: D 유지 + Space 짧게 누르기
- 종료 진행도: 약 6695.65px
- 사망: 0회

### 하단 대시 가시밭

- 실제 입력: D + Shift
- 종료 진행도: 약 7664.55px
- 대시 사용: 1회
- 사망: 0회

## 실행본 검증

GitHub에 반영한 `src/pass08.js`는 한 파일 안의 gzip 데이터를 한 번 해제해 실행한다. 외부 조각, 재귀 호출, 자동 재시도와 반복 생성 루프는 없다.

실제 GitHub용 압축 로더를 저장소와 같은 DOM 구조에 삽입해 검사했다.

- F8 입력 전: 8차 API 미생성
- 실제 F8 키 입력 후: 8차 API 생성
- 기존 캔버스 교체 성공
- 자동 감사: 16/16
- 콘솔 오류: 0건

이번 검사는 미로벽, 벽점프, 두 가시밭, 정밀 점프, 낮은 천장 점프와 붕괴석 곡선 이동을 각각 실제 브라우저에서 검사했다. 1차 시작점부터 8차 종료점까지 모든 구간을 한 번도 끊지 않고 완주하는 전체 통합 마라톤 검사는 아직 실시하지 않았다.
