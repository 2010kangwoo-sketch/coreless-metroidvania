# Coreless 새로운 40차 제작 — 8차

## 7차 연결 상태

8차는 7차 종료 체크포인트 `cp_mixed_end` 또는 기존 플레이어의 x좌표가 14500px 이상일 때 자동으로 시작한다. 기존 1~7차 실행본은 `src/main.js`에서 그대로 유지하고, 8차 전용 실행층이 그 위에서 이어진다. 수동 검사에서는 F8을 한 번 눌러 바로 진입할 수 있다.

## 추격 준비·가속 구간

- 기존 초대형 방 기준 범위: x=14700~18250
- 8차 내부 경로 길이: 3550px
- 장거리 추격전 진입점까지의 실제 진행 거리: 약 3165px
- 길이 갈라지지 않는 하나의 경사 경로
- 평지와 완만한 하강·상승 경사를 연속 연결
- 방향 화살표를 바닥선과 겹치지 않게 배치
- 입구 체크포인트: `cp_chase_prep`
- 종료 체크포인트: `cp_long_chase_entry`

## 세 단계 가속

1. 보폭 확보: x=650, 최소 전진 속도 4.2px/frame
2. 경사 가속: x=1480, 최소 전진 속도 5.0px/frame
3. 추격 진입: x=2320, 최소 전진 속도 5.6px/frame

플레이어는 일반 이동, 점프, 대시를 유지한다. 대시는 11프레임 동안 작동하고 재사용 대기시간은 96프레임이다.

## 장애물과 짧은 실전 추격

- 플레이어가 넘어야 하는 잔해 장애물: 5개
- 붕괴석 낙하 경고: 72프레임
- 짧은 실전 추격 길이: 약 910px
- 붕괴석 반지름: 78px
- 플레이어 크기: 34×48px
- 붕괴석 기본·적응 속도: 3.35~5.45px/frame

붕괴석은 장애물을 그대로 통과하는 것이 아니라 실제 충돌 판정을 거쳐 잔해를 파괴한다. 장애물에 부딪힐 때 속도가 0.92px/frame 감소하고, 24프레임 동안 감속 상태가 유지된다. 플레이어는 같은 장애물을 점프로 넘어야 하므로 두 대상의 한계와 역할이 다르다.

플레이어가 돌에 닿으면 압사 처리되고 추격 준비 체크포인트로 돌아간다. 낙하한 경우에도 체크포인트로 복귀한다. 추격이 시작된 뒤 카메라는 플레이어와 붕괴석의 중간을 기준으로 이동하여 두 대상이 가능한 한 같은 화면에 유지되도록 했다.

## 실제 Chromium 키보드 완주 검사

Chromium의 실제 키 입력 이벤트로 D를 계속 누르고, 장애물 접근 시 Space, 가속 지점에서 Shift를 입력했다.

- 실제 완주 시간: 약 9.10초
- 실제 점프 입력: 5회
- 실제 대시 입력: 4회
- 사망: 0회
- 압사: 0회
- 낙하: 0회
- 플레이어가 직접 넘은 장애물: 4개
- 붕괴석이 파괴한 장애물: 1개
- 장애물 충돌에 따른 붕괴석 감속: 1회
- 최종 로컬 x좌표: 3330px
- 최종 월드 x좌표: 18030px
- 최종 체크포인트: `cp_long_chase_entry`
- 브라우저 오류: 0건
- 자동 감사: 15/15 통과

## 실행 구조

`src/pass08.js`는 한 개의 파일이다. 내부 게임 코드를 gzip으로 압축해 포함하고, 페이지가 시작될 때 한 번만 압축을 해제하여 실행한다.

- 외부 데이터 조각 없음
- 재귀 호출 없음
- 자동 재시도 없음
- 반복 검증 루프 없음
- 7차 상태 확인용 250ms 폴링은 8차 활성화 여부만 확인하며, 활성화 뒤에는 아무 작업도 하지 않는다.
