## 기준선 및 환경

원격 15차 커밋에서 깨끗한 작업 사본을 만들었다. 첫 기준선 시도는 `/tmp` Playwright 런타임이 사라져 시작되지 않았고, 두 번째 시도는 Chromium 패키지의 `/tmp/fonts` 소유권 변경이 샌드박스에서 차단됐다. 저장소를 수정하지 않고 Chromium 138과 SwiftShader 라이브러리를 소유권 변경 없이 별도 경로에 풀었다.

복원 후 전체 검증은 소프트웨어 GPU에서 약 7fps로 제한됐다. 중간 두 실행은 게임 실패가 아니라 환경 성능 문제로 중단했으며, 불필요한 단일 프로세스·SwiftShader GPU 플래그를 제거한 동일 입력 검사로 끝까지 재실행했다.

## 구현

`src/v2/pass16-visuals.js`에 시각 표현 데이터와 검사를 분리했다.

- 10개 구역별 독립 팔레트
- 10개 구역별 고유 건축 모티프
- 원경 실루엣, 중경 건축, 경로 조명, 지형 스킨, 액터의 5개 계층
- 구역당 경로 조명 2개, 총 20개
- 충돌 디버그 계층 기본 비활성화
- 기존 바닥의 구역 번호를 시각 테마에 대응시키되 충돌 좌표는 변경하지 않음

구역 모티프는 시작 관문, 수직 우물, 매몰 아치, 압축 터널, 파괴 주조소, 거대 수로, 매달린 회랑, 공성 회랑, 심연 하강부, 파손 다리 순서다.

## 남은 한계

- 이번 차수는 시각 분리의 기반이며 최종 제작 그래픽은 아니다.
- 기존 회색박스 구조 프레임과 파괴 파편 일부는 후속 차수에서 전용 아트로 교체해야 한다.

- 장시간 추격 페이싱과 난이도는 사람 손 플레이로 추가 조정해야 한다.
- 전투, 최종 음향, 파괴 애니메이션과 시네마틱은 이번 범위가 아니다.
