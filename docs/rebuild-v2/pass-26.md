# Coreless Rebuild V2 — Pass 26

## 목적

25차에서 3번 초대형 방에 확정한 시각 지형과 충돌 지형의 분리 원칙을 전체 경로로 확장했다. 실제 충돌 바닥은 평평하고 정확하게 유지하면서 화면에는 암반·석조·목재·기계 재질의 불규칙한 외피가 보이도록 했다.

- 기준 로컬 커밋: `15aec94`
- 작업 브랜치: `rebuild/mega-room-40pass`
- 실행 버전: `rebuild-v2-pass26`
- `main` 병합: 없음
- 원격 게시: `gh` CLI 부재로 보류

## 25차 화면 감사

25차 전체 실행과 저장된 대표 화면을 기준선으로 확인했다. 3번 `BURIED RISE STRUCTURE`는 석조 블록·층리·빛·깊이 계층이 적용됐지만 나머지 구역은 굵은 직선 테두리와 일정한 깊이의 판 형태가 남아 있었다. 특히 붕괴 다리, 정밀 발판, 스프링 이후 구역이 서로 다른 재질이어야 함에도 같은 회색박스 문법으로 보였다.

## 구현

`src/v2/pass26-terrain.js`에서 실제 충돌 데이터로부터 비충돌 시각 외피를 생성했다.

- 원본 바닥 146개, 고유 바닥 145개 전부에 외피 적용
- 벽·천장·버팀벽 38개 전부에 외피 적용
- 바닥 재질 세부 단위 879개
- 벽체 패널·층리 세부 단위 242개
- 지형별 외피 모드
  - 자연 암반: 불규칙한 하부 윤곽과 층리
  - 매립 석조: 블록 이음새와 하부 보강
  - 산업·후기 기계 구역: 패널, 리벳, 모따기
  - 붕괴 다리: 개별 목재 판재와 나뭇결
  - 좁은 발판: 길쭉한 석주 형태
- 붕괴된 바닥의 점선 표시를 재질 파편 5개로 교체
- 시각 상단 흔들림을 충돌선 기준 최대 3px로 제한

각 외피는 원본 바닥·벽의 ID와 좌표를 직접 참조하며 `collisionChanges: 0`을 결정적 검사로 요구한다.

## 시각 확인과 증거 보정

시작 경사, 산업 구역, 거대 곡선, 장거리 하강, 붕괴 다리, 정밀 발판, 후기 구역의 7개 화면을 집중 캡처했다. 처음에는 장시간 실행 중 `canvas` 요소만 잘라 캡처할 때 HTML HUD와 캔버스 합성이 찢어져 제목 일부가 가려지는 현상이 있었다. 실제 페이지 계층 오류가 아니라 요소 캡처 방식의 문제였으므로, 렌더 프레임을 고정하고 전체 페이지를 캡처하도록 검사기를 수정했다. 보정된 캡처에서는 제목과 HUD가 정상 표시된다.

산업 구역의 넓은 빈 배경과 장거리 하강의 거대 실루엣은 27차 거대 구조물 그래픽 범위로 남겼다. 26차에서 임시 장식으로 덮지 않고 지형 외피 교체 범위만 유지했다.
