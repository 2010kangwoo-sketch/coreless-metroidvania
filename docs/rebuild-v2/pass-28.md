# Coreless Rebuild V2 — Pass 28

## 목적

27차까지의 Canvas 도형 중심 미술이 이전에 생성한 조각별 이미지의 질감·조명·깊이에 미치지 못한다는 문제를 인정하고, 실제 래스터 이미지 에셋을 게임 렌더러가 불러오는 제작 방식으로 전환했다. 28차는 전체 구역을 성급하게 덮는 차수가 아니라, 제철소 대표 장면 하나에서 목표 품질이 실제 게임 안에 구현될 수 있는지를 증명하는 품질 관문이다.

## 기준 이미지 상태

이전 업로드에서 1916×821 PNG 파일을 찾았지만 내부 데이터가 손상되어 픽셀을 읽을 수 없었다. 접근할 수 없는 파일을 확인한 것으로 간주하지 않았다. 대신 이전 대화에서 고정한 다음 조건과 27차 제철소의 실제 월드 구도를 결합해 28차 품질 목표 이미지를 제작했다.

- 어두운 폐허와 거대 기계 구조물
- 차가운 청록색 주변광과 제한된 황동·용광로 빛
- 빛줄기, 먼지, 안개와 거리 감쇠
- 작은 플레이어와 수 화면 높이 구조물의 크기 대비
- 복잡하지만 이동 경로가 읽히는 단일 연결 공간
- 기존 방 배치·높이 흐름·충돌의 임의 단순화 금지

원본 기준 이미지가 다시 확보되면 30차 승인 전에 화면 구성과 비례를 직접 대조해야 한다.

## 래스터 품질 목표 제작

27차 제철소 화면을 구도 입력으로 사용해 HUD와 도형 표현을 제거하고, 손으로 그린 2D 게임 배경에 가까운 고해상도 목표 이미지를 생성했다. 첫 생성본에 작은 가짜 플레이어가 포함돼 실제 플레이어와 중복되므로 두 번째 정밀 편집에서 그 실루엣만 제거했다.

최종 생성 지시의 핵심은 다음과 같다.

> 현재 제철소의 경사·천장·개방 공간·거대 곡선 지지대와 작은 캐릭터 비례를 유지하고, UI 없이 어두운 폐허형 기계 성소로 다시 그린다. 청록색 심연광, 제한된 황색 용광로 빛, 낡은 석조·산화 철·황동·그을음·사슬·먼지·안개를 사용하며, 평면 벡터 패널과 깨끗한 SF 표면을 피한다.

편집본은 1586×992 PNG로 보존하고, 게임에서는 111KB WebP로 변환해 사용한다.

## 구현

`src/v2/pass28-art-direction.js`에 이후 차수가 공유할 미술 기준과 에셋 계약을 추가했다.

- 색상 기준 9개
- 렌더 계층 5개: 원거리 배경, 중경, 래스터 백플레이트, 플레이 지형, 전경 분위기
- 변경 금지 조건 5개
- 30차 승인 관문
- 평면 임시 그래픽 허용 면적 최대 8%
- 화면당 거대 랜드마크 최소 1개
- 재질군 최소 4개
- 보이는 이미지 이음새 0개
- 충돌 변경 허용 0개

메인 모듈은 런타임 시작 전에 WebP를 비동기 로드하고 실제 크기를 확인한다. 제철소 월드 좌표 `(16480, 3050)`에 1586×992 크기로 배치하며, 카메라 밖에서는 그리지 않는다.

첫 통합 화면에서는 기존 26·27차 평면 지형이 이미지 위를 덮어 품질을 훼손했다. 렌더 순서를 수정해 해당 대표 구역의 도형 채움을 래스터 아래로 내리고, 위험 요소·상호작용 물체·얇은 충돌 경계·실제 플레이어만 이미지 위에 남겼다. 충돌 배열과 물리 데이터는 수정하지 않았다.
