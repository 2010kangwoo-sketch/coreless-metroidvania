## 21차 기준선

수정 전에 21차 전체 실제 키보드 검사를 다시 실행했다. 시작점부터 20차 출구까지 리셋·붕괴석 접촉·브라우저 오류 없이 통과했고, 이 결과를 22차 기준선으로 사용했다.

## 구현

`src/v2/pass22-level.js`에 13번째 후속 구역을 추가했다.

- 20차 출구 `(33700, 11160)`에서 22차 출구 `(29400, 11824)`로 이어지는 왼쪽 하강 동선
- 기존 20차 출구 경사와 착지 분지를 되짚은 뒤 시작되는 독립 충돌 구역
- 서쪽 착지대, 하단 선반, 출구 경사의 세 착지면
- 20차 입구보다 664px 낮은 최종 출구
- 새 지지 벽, 재질 색, 체크포인트와 전체 설계도 13구역 표기
- 세 착지면을 순서대로 밟아야 성립하는 완료 판정

## 실패와 수정 이력

1. 최초 배치에서는 새 입구 발판이 20차 출구 경사 아래에 겹쳤다. 위쪽 기존 경사가 충돌 대상으로 선택되어 첫 착지 조건이 영원히 성립하지 않았고, 검증기가 결과 파일 없이 반복됐다.
2. 첫 수정 뒤에는 20차 착지 분지가 새 첫 발판보다 위에서 플레이어를 받아 또다시 첫 발판을 건너뛰었다. 새 샤프트 전체를 착지 분지의 실제 왼쪽 끝 바깥으로 옮겼다.
3. 첫 전체 실행은 10차까지 통과했지만 검사기가 11차 세 번째 집게의 짧은 부착 가능 구간을 건너뛰어 낙하·리셋됐다. 런타임의 실제 부착 반경은 바꾸지 않고, 검사기가 접근 구간부터 실제 `E` 입력을 반복하도록 수정했다.
4. 첫 통과 화면에서 청사진 제목과 범례가 21차에 머물러 있었다. 13구역 경계·경로·제목을 22차로 갱신한 뒤 시작점부터 전체 검사를 다시 수행했다.

모든 실패 실행은 최종 완주에서 제외했다.
