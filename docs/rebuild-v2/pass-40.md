# Rebuild V2 · Pass 40

## 목적

40차는 새 이동 구간이나 장면을 추가하는 차수가 아니다. 1~39차에서 만든 초대형 단일 방을 하나의 완성 빌드로 고정하고, 사이트 주소에 접속했을 때 별도 명령 없이 게임으로 진입하는지 검증하는 최종 통합·배포 차수다.

이번 차수에는 사용자가 40차 이후로 미룬 배경 흐림, 형태 단순화, 캐릭터와 환경의 스타일 통합을 적용하지 않았다. 39차까지의 사실적 고밀도 방향을 그대로 보존했다.

## 구현

### 직접 진입과 자동 시작

- 사이트 루트에서 `/game/index.html`로 즉시 이동한다.
- 게임 문서가 열리면 11개 그래픽 자산 그룹을 병렬로 준비한다.
- 준비 상태를 `0/11`부터 `11/11`까지 화면에 표시한다.
- 준비가 끝나면 런타임을 자동 시작하고 캔버스에 키보드 포커스를 둔다.
- 실행 실패가 발생하면 빈 화면 대신 새로고침 안내를 표시한다.
- 작은 화면과 동작 감소 설정을 위한 반응형·접근성 규칙을 유지한다.

### 런타임 자산 최적화

기존 PNG 런타임 자산 49개를 원래 해상도와 알파 투명도를 유지한 고품질 WebP로 교체했다. 원본·생성 자료와 이전 증거 이미지는 `docs/rebuild-v2/assets/`에 그대로 보존했다.

- 런타임 자산 파일: `60`
- 최적화 전: `73,116,142 bytes` (`69.729 MiB`)
- 최적화 후: `16,009,222 bytes` (`15.268 MiB`)
- 감소율: `78.104%`
- 런타임 형식: `60/60 WebP`
- 외부 런타임 의존성: `0`

### 40차 감사 계층

`src/v2/pass40-release.js`에 직접 진입, 자동 시작, 포커스, 로딩·실패 화면, 자산 예산, 배포 포함·제외 범위, 후속 화면 보정 유예를 명시했다.

- 40차 공개 검증: `32/32`
- 릴리스 파일·경로 검증: `22/22`

- 충돌 변경: `0`
- 이동·전투 물리 변경: `0`
- 39차 그래픽 스타일 변경: `0`

## 플레이 검증 기준

39차 최종 실제 키보드 실행은 커밋 `c5ab950`에서 시작점부터 마지막 출구까지 통과했다.

- 실제 입력: `A/B/D/E/F/Shift/Space`
- 좌표 도우미: 사용하지 않음
- 최종 프레임: `23,020`
- 최종 위치: 약 `(24256.36, 12046.66)`
- 리셋: `0`
- 붕괴석 접촉: `0`
- 콘솔·페이지 오류: `0`

40차는 충돌·이동·전투 코드를 변경하지 않고 진입 화면과 자산 전송 형식만 바꿨다. 따라서 위 실행을 게임플레이 기준선으로 유지한다. 40차 전체 검사기와 GitHub 브라우저 워크플로는 새 제목, `98/98` 감사, `32/32` 릴리스 감사, WebP 자산, 로딩 해제 상태를 추가로 요구하도록 갱신했다.

내부 클라우드 브라우저 미리보기는 접근 권한이 거절되어 실행하지 않았다. 해당 권한을 다른 브라우저 방식으로 우회하지 않았다. 대신 Sites 생산 빌드와 산출물 검증을 통과시킨 뒤 실제 배포 상태가 `succeeded`인지 확인했다.
