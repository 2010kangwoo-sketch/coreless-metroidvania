## 23차 기준선

코드 변경 전에 23차 전체 실제 키보드 검사를 다시 실행했다. 시작점부터 23차 출구까지 22,004프레임에 통과했고 리셋·추격자 접촉·브라우저 오류는 모두 0이었다.

## 구현

`src/v2/pass24-integration.js`에 14구역 통합 규칙을 추가했다. 새 지형 구역은 만들지 않고 기존 경로의 기능 전환과 완료 조건을 통합했다.

- 전체 경로를 7개 목표로 재구성
  1. 기반 이동 경로
  2. 장거리 추격
  3. 집게·공중 대시
  4. 거대 곡선·붕괴 다리
  5. 정밀 발판·여진 붕괴
  6. 스프링·회복 샤프트
  7. 복합 건틀릿
- 이동, 발판, 추격, 파괴, 집게, 스프링, 전투의 7개 시스템 감사
- 현재 목표, 조작 힌트, 시스템별 완료 상태를 표시하는 우측 상단 HUD
- 목표 7/7과 시스템 7/7이 모두 성립할 때만 활성화되는 24차 후기 체크포인트
- 청사진의 `14 ZONES · 7 SYSTEMS · START → GOAL` 통합 표기
- 실제 입력 `A/B/D/E/F/Shift/Space` 요구

## 수정 이력

첫 전체 실행은 통합 조건을 모두 포함해 통과했다. 이후 출구 화면의 왼쪽 상태 패널이 23차 배경 경계에 일부 가려지는 기존 합성 문제를 시각 검사에서 발견했다. HTML 상태 패널과 조작 안내의 쌓임 순서를 월드 캔버스보다 명확히 높이고 불필요한 합성 변환을 제거했다. 집중 화면을 재검사한 뒤 시작점부터 전체 경로를 다시 실행해 최종 증거를 교체했다.
