## 22차 기준선

수정 전에 22차 집중 검사와 전체 실제 키보드 검사를 다시 실행했다. 시작점부터 22차 출구까지 리셋·붕괴석 접촉·브라우저 오류 없이 통과했고, 이 결과를 23차 기준선으로 사용했다.

## 구현

`src/v2/pass23-level.js`에 14번째 후속 구역 `CONVERGENCE GAUNTLET`을 추가했다.

- 22차 출구 `(29400, 11824)`에서 23차 출구 `(24120, 12110)`로 이어지는 왼쪽 진행 동선
- 실제 탑승 뒤에만 추격자를 시작하는 왕복 이동 발판 1개
- `F` 공격으로 각각 제거해야 하는 감시자 2개
- 기존 11차 집게 뒤에 이어지는 4·5번째 집게 앵커
- 전투와 집게 완료 뒤에만 발동하는 왼쪽 방향 스프링 1개
- 하부 낭떠러지, 추격 경로, 출구 경사와 23번째 후기 체크포인트
- 이동 발판 → 전투 → 집게 → 스프링 → 출구 순서를 모두 요구하는 완료 판정
- 전체 설계도 14구역과 복합 동작 범례

## 실패와 수정 이력

1. 최초 집중 실행에서는 22차의 서쪽 지지벽이 완료 뒤에도 충돌 대상으로 남아 23차 입구를 막았다. 22차 완료 시 해당 벽을 물리·렌더링 양쪽에서 제외했다.
2. 이동 발판을 기다리는 동안 추격자가 먼저 출발해 불공정한 접촉이 발생했다. 플레이어가 발판을 실제로 밟은 프레임부터 추격이 시작되도록 바꿨다.
3. 첫 집게를 너무 일찍 해제하면 두 번째 집게의 부착 반경에 닿지 않았다. 실제 `E` 입력 경로가 첫 집게의 하강 호를 충분히 사용하도록 검사기를 조정했다.
4. 공격 입력 큐를 처리 전에 지우는 초기화 순서 오류와 좁은 수직 공격 범위를 수정하고, 두 적을 제거하기 전에는 스프링이 발동하지 않도록 묶었다.
5. 첫 전체 실행은 23차 입구의 장식용 아치가 실제 충돌체로 남아 정지했다. 장식 아치를 충돌 목록에서 제거한 뒤 집중 검사를 재통과했다.
6. 다음 전체 실행은 실제 플레이와 23차 검사는 통과했지만, 기존 11차 검사가 누적 집게 횟수를 정확히 3회로만 요구해 실패했다. 기존 세 앵커 사용은 그대로 강제하면서 이후 앵커를 허용하도록 `>= 3`으로 호환성 검사를 보정했다.

모든 실패 실행은 최종 완주 증거에서 제외했다.

## 남은 한계

- 적과 공격은 피격·제거 순서를 검증하는 회색박스이며 최종 전투 AI, 체력, 애니메이션은 아니다.
- 추격자는 이 구역 전용 짧은 경로를 사용하며 기존 장거리 붕괴석과의 시각적 연결은 후속 아트 차수에서 보강해야 한다.
- 후기 체크포인트는 구간 완료 판정이며 영구 저장 시스템과 아직 연결되지 않았다.
- 자동 무중단 검사는 사람 플레이의 재미와 최종 난이도를 보장하지 않는다.
