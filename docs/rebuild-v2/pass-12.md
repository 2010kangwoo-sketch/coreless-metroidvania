# Rebuild V2 — 12차 공중 대시 가시밭

## 목표

12차는 11차 출구 `(22400, 6900)`에서 시작해 왼쪽으로 계속 하강하며, 점프만으로는 넘을 수 없고 점프 후 공중 대시를 사용해야 하는 긴 가시밭을 통과해 더 낮은 출구 `(20300, 7200)`에 도달하는 추격 회색박스다.

- 플레이어 흐름: 11차 출구 → 왼쪽 내리막 → 도약 경사 → 긴 가시밭 → 낮은 착지면 → 12차 출구 경사
- 붕괴석 흐름: 같은 구조물의 420px 폭 파괴 통로로 하강하며 가시 아래 구간을 통과한 뒤 출구 근처에서 다시 합류
- 필수 입력: `A` 이동 → `Space` 점프 → 공중 `Shift` 대시
- 표현 단계: 물리와 동선 검증을 위한 회색박스이며 최종 지형 그래픽은 넣지 않음

기준선 플레이에서는 기능 실패가 발견되지 않았다. 코드 점검에서는 런타임 생성자의 집게 사용 집합 중복 초기화와, 사망·수동 리셋 시 집게 상태 및 사용 집합이 명시적으로 초기화되지 않는 잠재 문제가 발견됐다. 중복 초기화를 제거하고 `resetPlayer()`가 집게 상태와 사용 집합까지 새로 만들도록 수정했다.

## 구현 내용

### 점프 단독과 공중 대시의 구분

| 항목 | 값 |
|---|---:|
| 가시밭 수평 폭 | 300px |
| 착지면 높이 차 | 입구보다 100px 낮음 |
| 계산상 체공 시간 | 약 43.52프레임 |
| 점프 단독 수평 이동 | 약 232.85px |
| 점프 단독 + 캐릭터 폭 | 약 266.85px |
| 점프·공중 대시 수평 이동 | 약 310.40px |
| 점프·공중 대시 + 캐릭터 폭 | 약 344.40px |

점프 단독은 약 33px 부족하고, 공중 대시를 사용하면 약 44px의 착지 여유가 생긴다. 완료 판정은 가시 구간 진입, 공중 이륙, 가시 근처에서 실제 공중 대시 사용, 반대편 착지, 낮은 출구 도달을 순서대로 요구한다.

## 발견한 문제와 수정

1. 이전 런타임에서 리셋할 때 집게 연결 상태와 이미 사용한 집게 ID 집합이 남을 수 있었다. 리셋 시 `createGrapple()`과 새 `Set`을 사용하도록 수정했다.
2. 생성자에 같은 집게 사용 집합 초기화가 중복돼 있었다. 중복 줄을 제거했다.
3. 첫 집중 검사 실행은 플레이어가 출구에 정상 도달했지만 검사 루프가 붕괴석 진행률 `0.98973`에서 끝나 완료로 기록되지 않았다. 이 실행은 성공으로 계산하지 않았다. 게임 조건은 바꾸지 않고 검사 프레임 예산만 늘린 뒤 다시 실행해 `0.99005`에서 통과했다.

## 남은 한계

- 플레이 가능한 충돌은 1~9구역까지이며, 10구역 낡은 나무다리 피날레는 아직 청사진 데이터다.
- 가시 충돌은 현재 회색박스 단계의 연속 위험 띠 판정이다. 개별 가시 삼각형 단위의 세부 판정은 후속 난이도 조정 대상이다.
- 구조물, 지지대, 파편, 먼지와 화면 흔들림은 임시 표현이다.
- 전투, 적, 터렛, 피해, 음악, 효과음, 시네마틱 카메라와 최종 그래픽은 이번 차수 범위가 아니다.
- 자동 무중단 통과는 확인했지만, 사람 기준 15~20분 플레이 시간과 최종 재미를 검증한 것은 아니다.
