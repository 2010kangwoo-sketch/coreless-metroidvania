# Coreless Rebuild V2 — Pass 19

## 목적

18차의 좁은 정밀 발판 구역을 그대로 유지하면서, 추격 종료의 충격이 뒤늦게 전달되어 플레이어가 떠난 발판부터 순서대로 붕괴하는 후속 파괴 구간을 추가했다. 19차는 19~24차 추격·파괴 단계의 첫 작업이다.

- 기준 원격 커밋: `4a53a4841746b524aeb744b383772b9d5d1b5781`

## 18차 기준선과 검사기 수정

수정 전 실제 Chromium 기준선에서 게임 오류나 콘솔 오류는 없었지만, 검사기가 11차 세 번째 집게를 프레임 시간에 의존해 다루면서 출구를 놓치고 리셋됐다. 집게 부착 가능 거리를 런타임의 실제 `attachRadius`와 일치시키고, 세 번째 해제 뒤 고정 입력 대신 플레이어 위치·속도 피드백으로 출구 중심을 향하도록 수정했다.

수정한 검사기로 시작점부터 18차 출구까지 좌표 도우미 없이 다시 통과한 뒤 19차를 구현했다. 이 실패는 게임 완주 실패가 아니라 기존 전체 검사기의 입력 타이밍 회귀로 기록한다.

## 구현

`src/v2/pass19-level.js`에 18차 발판 위에서 작동하는 후속 붕괴 데이터를 추가했다.

- 18차 좁은 발판 12개와 일대일로 연결된 파괴 지지대 12개
- 실제 착지 시 해당 발판을 무장하고, 플레이어가 떠난 뒤에만 붕괴하는 안전 규칙
- 기본 24프레임과 0~3단계 충격 파동 오프셋을 합친 지연 붕괴
- 발판 12개, 지지대 12개, 잔해 분출 12회의 완료 감사
- 다리 피날레에서 봉인된 붕괴석의 충격을 이어받는 `AFTERSHOCK COLLAPSE` 상태
- 12개 붕괴 뒤 18차 출구 체크포인트를 안정화하는 완료 판정
- 붕괴 대기 경고선, 균열 지지대, 회색박스 잔해 렌더링
- 전체 설계도에 12개 후속 붕괴 지점과 19차 구역 표기

붕괴석은 15차 다리 아래에서 계속 봉인되며 18차 구역으로 다시 진입하지 않는다. 지형과 충돌 경계는 18차와 동일하게 유지했다.

## 집중 검사

집중 검사는 17차 최종 착지대 배치에만 좌표 도우미를 사용하고, 이후에는 실제 `D/Space` 키 이벤트로 18차와 19차를 통과했다. 좌표 도우미를 썼으므로 전체 완주로 계산하지 않았다.

- 실제 입력: `D`, `Space`
- 좁은 발판 착지/무장/붕괴: 12/12/12
- 잔해 분출: 12회
- 최대 동시 붕괴 대기: 2개
- 리셋/붕괴석 접촉: 0/0
- 런타임/19차 감사: 28/28, 30/30
- 콘솔/페이지 오류: 0/0
- 결과: 통과

첫 종료 캡처는 완료 직후 화면 흔들림 중 저장되어 제목이 찢어진 것처럼 보였다. UI 계층 오류가 아니라 캡처 시점 문제였으며, 완료 후 500ms를 기다려 다시 저장한 화면에서 정상 렌더링을 확인했다.

## 전체 검사 실패와 수정

최종 증거를 만들기 전 전체 경로를 반복 실행하며 다음 실패를 별도로 기록했다.

1. 18차 기준선에서 세 번째 집게의 입력 타이밍이 프레임률에 의존해 출구를 놓쳤다. 실제 부착 반경과 플레이어 상태를 기준으로 검사 조건을 바꿨다.
2. 설계도 표기만 고친 뒤 수행한 전체 재검사에서는 세 번째 집게 해제 후 고정 `D` 입력이 연속 진입 속도와 겹쳐 출구 오른쪽으로 지나쳤다. 목표 X좌표와 현재 속도에 따라 `A/D`로 제동하는 피드백 제어로 바꿨다.

두 실행 모두 완주로 계산하지 않았으며, 수정 뒤 시작점부터 다시 실행했다.

## 남은 한계

- 균열, 잔해와 파괴 지지대는 충돌·페이싱 검증용 회색박스다.
- 다리 충격에서 후속 붕괴로 이어지는 연출은 상태와 화면 흔들림만 있으며 최종 애니메이션, 입자, 음향과 시네마틱은 없다.
- 붕괴 지연과 좁은 발판의 사람 손 기준 난이도는 추가 조정이 필요하다.
- 체크포인트 안정화는 구간 완료 판정이며 영구 저장 시스템과 연결되지 않았다.
- 자동 무중단 검사가 사람 플레이의 재미와 15~20분 목표 시간을 보장하지는 않는다.
