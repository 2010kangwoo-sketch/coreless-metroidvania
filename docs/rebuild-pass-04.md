# Coreless 새로운 40차 제작 — 4차

## 선행 확인: 3차 실제 화면

3차 게임을 실제 Canvas와 Chromium으로 다시 실행해 다음 문제를 확인했다.

- 수직축 카메라가 x=3300에 있어 이전 정밀 점프 구역의 가시가 왼쪽 가장자리에 노출됨
- 수직축 자체가 화면 중앙보다 왼쪽에 치우쳐 이동 경로가 불명확함
- 상단 체크포인트와 다음 공간을 연결할 오른쪽 출구가 구조적으로 막혀 있음
- 상단 발판과 체크포인트가 가까워 캐릭터가 겹쳐 보일 가능성이 큼

## 4차 수정

- 수직축 카메라 x를 3450으로 조정해 축을 화면 중앙에 배치
- 수직축 양옆을 어둡게 처리해 이전 구역과 빈 공간 노출을 줄임
- 상단 오른쪽 벽을 y=-460 위에서만 유지해 실제 출구를 개방
- 집게 횡단 체크포인트를 상단 출구와 분리해 배치
- 기존 벽 미끄러짐·벽점프 기능을 그대로 유지하고 420프레임 완주 재검증

## 집게·클로 횡단 구현

- 네 번째 공간: x=4500~6900, y=-900~-180
- 입구와 착지 발판 사이 1340px 낭떠러지
- 대체 발판 없이 다섯 집게만 이어지는 단일 공중 경로
- 집게 간격 270px
- 줄 길이 155~180px
- 잡기 범위 126px
- E: 가까운 집게 잡기/해제
- A/D: 진자 각가속도 제어
- Space: 집게 해제와 1.8px/frame 상향 보정
- 최대 각속도 0.075rad/frame
- 실제 해제 속도 최대 약 13.5px/frame
- 낭떠러지 가시 접촉 시 입구 체크포인트 복귀
- 입구·종료 체크포인트와 상단 수평 추적 카메라

## 실제 Chromium 키보드 검증

플레이 테스트용 순간이동 후 이동 자체는 Chromium의 실제 키보드 이벤트만 사용했다.

- D를 실제로 누른 상태에서 입구 발판 끝까지 이동
- Space로 점프
- 각 집게 접근 시 E를 실제로 눌러 연결
- A/D로 진자를 가속
- 오른쪽 각도 약 0.83~0.89rad에서 Space로 해제
- G1 → G2 → G3 → G4 → G5 순서로 모두 연결
- 낙하·피해·사망 0회
- 종료 체크포인트 `cp_grabber_end` 도달
- 최종 x 약 6613.8px
- 브라우저 콘솔 오류 0건
- 자동 감사 12/12 통과
